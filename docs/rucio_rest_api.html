parse_scope_name

parse_scope_name(scope_name, vo)

Parses the given scope_name according to the schema's SCOPE_NAME_REGEXP and returns a (scope, name) tuple.

Arguments:

  • scope_name: the scope_name string to be parsed.
  • vo: the vo currently in use.

Raises:

  • ValueError: when scope_name could not be parsed.

Returns:

a (scope, name) tuple.

try_stream

try_stream(generator, content_type=None) -> "flask.Response"

Peeks at the first element of the passed generator and raises an error, if yielding raises. Otherwise returns a flask.Response object.

Arguments:

  • generator: a generator function or an iterator.
  • content_type: the response's Content-Type. 'application/x-json-stream' by default.

Returns:

a response object with the specified Content-Type.

generate_http_error_flask

generate_http_error_flask(status_code: "int", exc: "Union[str, BaseException]", exc_msg: "Optional[str]" = None, headers: "Optional[HeadersType]" = None) -> "flask.Response"

Utitily function to generate a complete HTTP error response.

Arguments:

  • status_code: The HTTP status code to generate a response for.
  • exc: The name of the exception class or a RucioException object.
  • exc_msg: The error message.
  • headers: any default headers to send along.

Returns:

a response object representing the error.

class CodeOIDC(ErrorHandlingMethodView)

IdP redirects to this endpoint with the AuthZ code Rucio Auth server will request new token. This endpoint should be reached only if the request/ IdP login has been made through web browser. Then the response content will be in html (including the potential errors displayed). The token will be saved in the Rucio DB, but only Rucio code will be returned on the web page, or, in case of polling is True, successful operation is confirmed waiting for the Rucio client to get the token automatically.
